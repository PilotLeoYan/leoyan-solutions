## рџљЂ Quick Start

### 1 - Fork / clone

```bash
git clone https://github.com/PilotLeoYan/leoyan-solutions.git
cd leoyan-solutions
```

### 2 - Install dependencies

```bash
pip install -r requirements.txt
```

### 3 - Update your username

In `mkdocs.yml`, replace every `PilotLeoYan` and `YOUR_CF_HANDLE`:

```yaml
site_url: https://PilotLeoYan.github.io/leoyan-solutions/
repo_url: https://github.com/PilotLeoYan/leoyan-solutions
```

### 4 - Serve locally

```bash
mkdocs serve
```

Open [http://127.0.0.1:8000](http://127.0.0.1:8000).

## рџ“„ Adding a Problem

1. Copy the template:

   ```bash
   cp docs/problems/_template.md docs/problems/999X-my-problem.md
   ```

2. Edit the **YAML frontmatter**:

   ```yaml
   ---
   title: "999X вЂ“ My Problem"
   problem_id: "999X"
   problem_url: "https://codeforces.com/problemset/problem/999/X"
   difficulty: 1400
   date_solved: "2025-01-01"
   tags:
     - binary search
     - greedy
   ---
   ```

3. Write your explanation using any of:

   - `=== "Python"` / `=== "C++17"` tabbed code blocks
   - ` ```mermaid ` diagrams
   - `!!! note "Complexity Analysis"` admonitions
   - `$O(n \log n)$` inline LaTeX

4. Add it to the **`nav`** section in `mkdocs.yml`.

5. Push:

   ```bash
   git add docs/problems/999X-my-problem.md mkdocs.yml
   git commit -m "feat: add 999X My Problem"
   git push
   ```

GitHub Actions auto-builds and deploys in ~60 seconds.
